Conversation — the primary interface (O11) A real agentic chat whose only hands are the nine typed engine tools. It proposes anything, commits only through confirm_task / run_loop / resolve_drift / swap_adapter. Verdicts and drift post into it via the event bus — the system talks back unprompted.
Truth — where claims are verified at a glance The live verdict with its reasons, bound to the exact diff SHA it judged. The acceptance rail is O12's behavioral axis: real checks (web@1, contract@1) with real evidence — this panel renders the actual run store, never a model's opinion. The gate has no seat.
Stage — preview-as-evidence Every attempt gets its own sandbox and its own live preview; what you see is bound to the verdict. Click through a flow and mark it “must keep working” — it becomes an executable acceptance check attached to the spec.
